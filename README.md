## How it works (matches your assignment steps)

**Step 1 — Parsing**
- Tab "Upload & Parse": upload multiple PDFs, they get parsed with LlamaParse.
- Tab "Parsed Text": view the full parsed text of any uploaded PDF.

**Step 2 — Chunking + RAG**
- Tab "Chunks": adjust chunk size/overlap, generate chunks, view every chunk.
  This also builds a vector store (Chroma) using Gemini embeddings.
- Tab "Chat (RAG)": ask a question. You'll see:
  - the chunks retrieved from the vector store (with similarity distance)
  - the final answer, generated by Gemini using ONLY those retrieved chunks

**Login**
- First run auto-creates `users.json` with `admin` / `admin123`.
- You can sign up new accounts from the "Sign up" tab.

## Notes
- Vector store is in-memory (`chromadb.Client()`), so it resets each time you
  restart the app — regenerate chunks after restarting.
- Embedding model: `models/text-embedding-004` (Gemini). Chat model: `gemini-2.0-flash`.
  Change `EMBED_MODEL` / `CHAT_MODEL` in `rag_engine.py` if your professor wants a different model.
